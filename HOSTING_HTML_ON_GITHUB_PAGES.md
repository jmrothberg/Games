# Hosting an HTML file on GitHub Pages

Instructions for OpenCoder (or any agent) to publish any HTML file from this repo as a shareable, playable link.

## The short version

Any `.html` file committed to the `main` branch is automatically served at:

```
https://jmrothberg.github.io/Games/<path/to/file.html>
```

Example — `Generated_games/space_invaders_animated.html` is live at:
<https://jmrothberg.github.io/Games/Generated_games/space_invaders_animated.html>

## One-time setup (already done for this repo)

GitHub Pages is enabled in legacy / deploy-from-branch mode, serving the root of `main`. Config is:

- Build type: `legacy`
- Source branch: `main`
- Source path: `/`

To replicate on a new repo:

```bash
gh api repos/<owner>/<repo>/pages \
  --method POST \
  -f build_type=legacy \
  -f 'source[branch]=main' \
  -f 'source[path]=/'
```

If Pages was already configured differently, use `--method PUT` instead of `POST`.

## Publishing a new HTML file — the workflow

1. **Drop the file** anywhere under the repo (e.g. `Generated_games/my_new_game.html`). Single-file HTML only — all JS/CSS/assets must be inlined or loaded from CDN. Relative `src="./sprite.png"` paths work if the asset is committed alongside.

2. **Commit and push** to `main`:

   ```bash
   git add Generated_games/my_new_game.html
   git commit -m "Add my_new_game"
   git push
   ```

3. **Wait ~30-60s** for Pages to rebuild. Check status:

   ```bash
   gh api repos/jmrothberg/Games/pages/builds 2>&1 | head -20
   ```

   Or force a rebuild immediately:

   ```bash
   gh api repos/jmrothberg/Games/pages/builds --method POST
   ```

4. **Verify the URL is live**:

   ```bash
   curl -sI https://jmrothberg.github.io/Games/Generated_games/my_new_game.html | head -1
   # Expect: HTTP/2 200
   ```

5. **Add a row to the README** games table. Open `README.md`, find the `## Play Games Generated by CodeRunner` table, and append:

   ```markdown
   | **My New Game** | [Play](https://jmrothberg.github.io/Games/Generated_games/my_new_game.html) | <model-name> |
   ```

6. **Commit the README update** and push. The link is now shareable.

## URL rules

- Spaces in filenames → `%20` in the URL. Prefer underscores to avoid this.
- Filenames are case-sensitive on GitHub Pages (even though the Pages lookup is loose — don't rely on it).
- Files named `index.html` in a folder become the folder's default page: `Generated_games/` → `Generated_games/index.html`.

## Troubleshooting

- **404 after push**: Pages is still building. Run `gh api repos/jmrothberg/Games/pages/builds | head -20` — if the newest entry isn't `"status":"built"` yet, wait. If it says `"errored"`, read the `error.message` field.
- **Blank page / JS errors**: Open browser devtools. Almost always a relative asset path that doesn't exist, or a `file://`-only API (e.g. loading local JSON via `fetch`). Inline the asset or fix the path.
- **Old version still served**: Pages CDN caches aggressively. Hard-refresh (Cmd-Shift-R), or append a cache-buster `?v=2` to the URL while testing.
- **Pages config drift**: `gh api repos/jmrothberg/Games/pages` shows the current config. If someone flipped it to `workflow` mode without a workflow file, flip it back with the `gh api ... --method PUT` command above.

## Quick reference — the whole flow in one block

```bash
# From the repo root, after dropping a new HTML file:
FILE="Generated_games/my_new_game.html"
URL="https://jmrothberg.github.io/Games/$FILE"

git add "$FILE" README.md
git commit -m "Add $(basename $FILE .html)"
git push

gh api repos/jmrothberg/Games/pages/builds --method POST
echo "Play at: $URL"
```
